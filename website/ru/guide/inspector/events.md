# Инспектор слушателей событий

Просмотр всех зарегистрированных слушателей событий в вашем приложении — организованных по классу события.

![Инспектор слушателей событий](/images/inspector/events.png)

## Что отображается

| Поле | Описание |
|------|----------|
| Класс события | Полное имя класса события |
| Слушатели | Список классов/методов слушателей, подписанных на это событие |
| Группы | Общий, Web и Console диспетчеры событий (разделены вкладками) |

## Вкладки

- **Общие** — Слушатели событий, общие для всех контекстов
- **Web** — Слушатели, специфичные для обработки HTTP-запросов
- **Console** — Слушатели, специфичные для выполнения CLI-команд

## Эндпоинты API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/inspect/api/events` | Все слушатели событий по группам |

## Как это работает

Инспектор читает конфигурацию диспетчера событий приложения и перечисляет всех зарегистрированных слушателей. Для Symfony сюда входят события ядра (`kernel.request`, `kernel.response` и т.д.), события Doctrine и пользовательские события приложения.
