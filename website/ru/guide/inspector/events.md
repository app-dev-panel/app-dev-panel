# Инспектор слушателей событий

Просмотр всех зарегистрированных слушателей событий в вашем приложении — с группировкой по классу события.

![Инспектор слушателей событий](/images/inspector/events.png)

## Что показывает

| Поле | Описание |
|------|----------|
| Event class | Полное имя класса события (FQCN) |
| Listeners | Список классов/методов слушателей, подписанных на это событие |
| Groups | Диспетчеры событий Common, Web и Console (разделены по вкладкам) |

## Вкладки

- **Common** — Слушатели событий, общие для всех контекстов
- **Web** — Слушатели, специфичные для обработки HTTP-запросов
- **Console** — Слушатели, специфичные для выполнения CLI-команд

## API-эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/inspect/api/events` | Все слушатели событий по группам |

## Как это работает

Инспектор считывает конфигурацию диспетчера событий приложения и перечисляет всех зарегистрированных слушателей. Для Symfony это включает события ядра (`kernel.request`, `kernel.response` и т.д.), события Doctrine и пользовательские события приложения.
