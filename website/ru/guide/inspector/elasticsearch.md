# Инспектор Elasticsearch

Проверка состояния кластера Elasticsearch, просмотр индексов и выполнение поисковых запросов.

## Функции

| Функция | Описание |
|---------|----------|
| Состояние кластера | Статус green/yellow/red |
| Список индексов | Все индексы с количеством документов и размерами |
| Детали индекса | Маппинги, настройки и статистика |
| Поиск | Выполнение поисковых запросов к индексу |
| Произвольный запрос | Выполнение любых запросов к API Elasticsearch |

## Поиск

Выполнение запросов Elasticsearch к любому индексу. Укажите индекс, тело запроса, лимит и смещение. Результаты отображаются с исходными данными документов.

## Произвольный запрос

Для расширенного использования выполняйте любые запросы к API Elasticsearch, указав HTTP-метод, эндпоинт и опциональное тело запроса.

## Эндпоинты API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/inspect/api/elasticsearch` | Состояние кластера + список индексов |
| GET | `/inspect/api/elasticsearch/{name}` | Маппинги, настройки, статистика индекса |
| POST | `/inspect/api/elasticsearch/search` | Поисковый запрос к индексу |
| POST | `/inspect/api/elasticsearch/query` | Произвольный запрос к API Elasticsearch |

**Тело запроса поиска:**
```json
{
    "index": "products",
    "query": {"match": {"name": "widget"}},
    "limit": 10,
    "offset": 0
}
```

**Тело произвольного запроса:**
```json
{
    "method": "GET",
    "endpoint": "/_cluster/stats",
    "body": null
}
```

## Требования

Требуется реализация <class>AppDevPanel\Api\Inspector\Elasticsearch\ElasticsearchProviderInterface</class> в DI-контейнере. Каждый адаптер предоставляет собственную реализацию на основе используемой клиентской библиотеки Elasticsearch.
