# Поток данных

На этой странице описано, как отладочные данные проходят от вашего приложения к панели ADP.

## Обзор

```
Приложение → Адаптер → Прокси → Коллекторы → Debugger → Хранилище → API → Фронтенд
```

## Пошаговый процесс

### 1. Установка адаптера

Ваше приложение работает с установленным адаптером ADP (например, адаптер Symfony или Laravel). Адаптер регистрирует себя в DI-контейнере фреймворка во время загрузки.

### 2. Регистрация прокси

Адаптер регистрирует **прокси**, оборачивающие PSR-интерфейсы в DI-контейнере. Например, `LoggerInterfaceProxy` оборачивает ваш PSR-3 логгер. Код приложения продолжает использовать стандартные интерфейсы -- он не знает о перехвате.

### 3. Сбор данных

Во время обработки запроса прокси перехватывают вызовы и передают данные **коллекторам**:

- Вызовы логгера идут в `LogCollector`
- Отправленные события идут в `EventCollector`
- HTTP-запросы клиента идут в `HttpClientCollector`
- SQL-запросы идут в `DatabaseCollector` (через хуки адаптера)

Каждый коллектор накапливает данные в памяти на протяжении запроса.

### 4. Сброс Debugger

Когда запрос завершается (или консольная команда заканчивается), **Debugger** инициирует завершение:

1. Вызывает `shutdown()` на всех коллекторах
2. Вызывает `getCollected()` для получения накопленных данных
3. Сериализует объекты через `Dumper` (с контролем глубины и обнаружением циклических ссылок)
4. Вызывает `flush()` на хранилище

### 5. Сохранение в хранилище

**FileStorage** записывает три JSON-файла на каждую отладочную запись:

| Файл | Содержимое |
|------|------------|
| `{id}.summary.json` | Метаданные записи (время, URL, статус, коллекторы) |
| `{id}.data.json` | Полные данные коллекторов |
| `{id}.objects.json` | Сериализованные PHP-объекты |

### 6. Отображение во фронтенде

**API** предоставляет сохранённые данные через REST-эндпоинты. Фронтенд использует **SSE** (Server-Sent Events) для обнаружения новых записей в реальном времени:

1. Фронтенд подписывается на `/debug/api/event-stream`
2. API опрашивает хранилище каждую секунду, вычисляя MD5-хеш сводок
3. При появлении новой записи хеш меняется и отправляется событие
4. Фронтенд загружает обновлённый список записей

## Внешние приложения

Не-PHP приложения могут отправлять отладочные данные через **Ingestion API** (`POST /debug/api/ingest`), минуя пайплайн прокси/коллекторов. Данные записываются напрямую в хранилище.
