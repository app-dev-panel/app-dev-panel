Что если ваш AI-ассистент мог бы смотреть на данные отладки и помогать исправлять баги? С MCP-сервером ADP это именно так и работает.

## Что такое MCP?

**MCP (Model Context Protocol)** — это стандарт подключения AI-ассистентов к внешним источникам данных. Вместо копирования сообщений об ошибках в чат, AI-ассистент может запрашивать ADP напрямую.

## MCP-интеграция ADP

ADP предоставляет данные отладки через два MCP-транспорта:

### stdio-транспорт

Для локальной разработки с CLI-инструментами:

```json
{
    "mcpServers": {
        "adp": {
            "command": "php",
            "args": ["vendor/bin/adp", "mcp:stdio"]
        }
    }
}
```

### HTTP-транспорт

Для удалённых или веб-инструментов:

```
POST /debug/api/mcp
Content-Type: application/json

{"jsonrpc": "2.0", "method": "tools/list", "id": 1}
```

## Доступные инструменты

MCP-сервер предоставляет AI-ассистентам следующие инструменты:

| Инструмент | Описание |
|-----------|----------|
| `list_entries` | Список последних записей отладки с фильтрами |
| `get_entry` | Полные данные конкретной записи |
| `get_collector_data` | Данные конкретного коллектора записи |
| `search_logs` | Поиск по сообщениям лога |
| `get_exceptions` | Список пойманных исключений со стек-трейсами |
| `get_queries` | Список SQL-запросов с временем выполнения |
| `get_http_requests` | Список исходящих HTTP-запросов |
| `inspect_routes` | Просмотр маршрутов приложения |
| `inspect_config` | Просмотр конфигурации приложения |
| `inspect_container` | Просмотр сервисов DI-контейнера |

## Пример из практики

Представьте, что вы видите ошибку 500. С MCP-интеграцией можно спросить AI:

> «Посмотри последние записи отладки и выясни, почему последний запрос вернул 500»

AI выполнит:

1. `list_entries` — найдёт записи со статусом 500
2. `get_collector_data` — изучит коллектор исключений
3. `get_queries` — проверит, не упал ли SQL-запрос
4. Предоставит диагноз с полным контекстом

```
AI: Нашёл проблему. Запрос к /api/users упал, потому что
миграция таблицы `users` не была выполнена. DatabaseCollector
показывает ошибку "relation 'users' does not exist" на запросе:

  SELECT * FROM users WHERE active = true

Запустите `php artisan migrate` для исправления.
```

## Безопасность

MCP-эндпоинты используют ту же IP-фильтрацию, что и остальной API. По умолчанию доступ только с localhost. Для production настройте `auth_token`.

## Настройка

1. Установите ADP с адаптером для вашего фреймворка
2. Укажите MCP-настройки вашего AI-инструмента на ADP
3. Начните отлаживать с помощью AI

Подробные инструкции в [руководстве по MCP-серверу](/ru/guide/mcp-server).
